# SZU_Unlimited_Water_Card
> 将深圳大学校园卡的 UID 克隆到可写 CUID 卡上，实现无限热水。  
> 基于 Arduino + RFID-RC522 搭建。

---
##  项目背景

学校热水系统仅通过卡片的 **UID**（唯一标识符）进行身份验证 —— **不校验任何加密签名数据**。这意味着只需将校园卡的 UID 克隆到一张可写的 **CUID 卡**上，就能骗过系统。

> ⚠️ **本项目揭露了深大热水系统的真实安全隐患。**  
> 希望能引起学校重视，早日升级到更安全的认证系统。

**特别鸣谢**本仓库根据SZU_FREE_HOT_WATER_Card修改而来，感谢作者Suyixiu提供的宝贵思路，原仓库地址：https://github.com/Suyixiu/SZU_FREE_HOT_WATER_CARD

> **本版改进：** 正确计算 BCC（块校验字），修正 SAK 字节（`0x08`，MIFARE Classic 1K），修正 ATQA 字节（`0x04 0x00`）—— 修复了绝大部分卡片写入失败的问题。

##  软件准备

### 1. 安装 Arduino IDE

### 2. 安装 RFID 库

**两种方式任选其一：**

**方式 A —— 使用仓库自带的 `RFID.zip`（推荐新手）**
1. Arduino IDE 中点击：**项目 → 加载库 → 添加 .ZIP 库…**
2. 选择本仓库中的 `RFID.zip`

**方式 B —— 使用 MFRC522 库（不推荐，）**
1. Arduino IDE 中点击：**工具 → 管理库…**
2. 搜索 `MFRC522`（作者 GithubCommunity）并安装

### 3. 烧录程序

1. 用 Arduino IDE 打开 `SZU_FREE_HOT_WATER_CARD/SZU_FREE_HOT_WATER_CARD.ino`
2. 选择开发板：**工具 → 开发板 → Arduino Uno**
3. 选择端口：**工具 → 端口**
4. 点击 **上传**（→ 箭头）
5. 打开 **串口监视器**（波特率设为：**9600**）

## 🚀 使用步骤

### 第一步 —— 克隆校园卡 UID

1. 将**校园卡**放在 RC522 模块上
2. 串口监视器会打印：`[CAMPUS] UID: XXXXXXXXX`
3. **3 秒内**拿开校园卡，放上 **CUID 卡**
4. 写入成功后显示：`Write card OK!`，然后 `SUCCESS!`（可能会出现 The card may be bricked，这是正常的，以能否出热水为准）
5. 按一下 Arduino 的 **Reset** 键验证 —— CUID 卡的 UID 应该和校园卡一致

### 第二步 —— 热水机上使用

1. 先用**校园卡**在热水机上刷一下 → 听到"B"的一声，屏幕显示 `1.25`
2. 在 `1.25` 数字没消失前（或是0.05没有跳转到下一个数字前，总之时机你自己把握），**拿开校园卡**
3. 立刻把 **CUID 卡**放上去就ok了
4. 余额用完时，重复步骤 1-3 即可继续使用

## ❓ 常见问题

<details>
<summary><b>Q: 能不能用这个改饭卡余额？</b></summary>

**不能。** 本项目只克隆了卡片的 **UID**（序列号）。饭卡余额储存在卡片的加密扇区里，有密钥保护，无法读取和修改。不要尝试。
</details>

<details>
<summary><b>Q: CUID 卡进图书馆后不能用了？</b></summary>

深大图书馆闸机会写入卡片的 Block 0，可能会**锁死** CUID 卡（变成只读）。之后这张卡在热水机上可以直接刷出 1.25，不再需要前面的刷卡步骤。但锁死后无法再重新写入。
</details>

<details>
<summary><b>Q: 串口显示 "Auth FAILED" 怎么办？</b></summary>

多半是你用了**普通 UID 卡**而不是 **CUID 卡**。只有 CUID 卡允许写 Block 0。淘宝搜"CUID 可写卡"，不要买成普通 MIFARE 卡（厂家的 UID 是锁死的）。
</details>
